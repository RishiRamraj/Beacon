# ADR 0009: Licence is GPLv3

- **Status:** Accepted
- **Date:** 2026-07-20

## Context

Beacon statically links bsnes-jg ([ADR 0002](0002-embed-bsnes-jg-directly.md)), which is
GPLv3. Static linking makes the whole binary GPLv3. This is a consequence of the emulator
choice rather than an independent decision, but it has practical obligations worth recording
and it constrains what the project can become later.

The alternative bases do not escape it. bsnes-plus is GPLv2, MesenCE is GPL-3.0, and snes9x's
licence is genuinely non-commercial-only, so it is neither OSI-approved nor GPL-compatible.

## Decision

**GPLv3. Decided and closed.** Accepted rather than worked around.

## Rationale

- Forced by static linking of bsnes-jg, and not a constraint worth fighting.
- Every serious candidate core is GPL anyway, so the licence was a non-differentiator in the
  emulator comparison rather than a reason to pick differently.
- The dependency list is already compatible: **Apache-2.0 is one-way compatible into GPLv3**,
  so Steam Audio is fine, and Lua's MIT is fine. Nothing currently in the dependency list
  conflicts.
- Piper's relicence from MIT to GPL-3.0 (on the move to `OHF-Voice/piper1-gpl`) is harmless
  here for the same reason.
- `sameboy-sys` is a GPLv3 Rust crate compiling an emulator core into the crate, so it is a
  useful precedent for the licensing shape as well as the build pattern.

The project is legally clean beyond the licence too: **no ROM data ships**, no circumvention,
and no redistribution of anything we do not have the right to redistribute. ROMs are
user-supplied and identified by SHA-1 ([ADR 0004](0004-plugin-model-toml-profile-plus-lua.md)).

## Consequences

Obligations to honour:

- Ship or offer the complete corresponding source, **including our bsnes-jg patch if we ever
  write one** ([ADR 0010](0010-defer-emulator-hook-patch.md)), and including the build scripts
  needed to reproduce the binary.
- Keep upstream copyright notices intact.
- Carry `LICENSE` plus a `THIRD-PARTY.md` naming bsnes-jg (GPLv3), Lua (MIT), and Steam Audio
  (Apache-2.0).

Costs accepted:

- **snes9x stays permanently out of the tree.** Its non-commercial clause is GPL-incompatible,
  so it cannot be linked even optionally.
- **This rules out a permissive fork later.** If that ever becomes desirable, the base would
  have to change to ares (ISC), which is part of why the alternatives comparison stays
  documented in [ADR 0002](0002-embed-bsnes-jg-directly.md).
- Piper is ruled out for anyone wanting to build a permissive fork.
- On the upside, GPLv3 is also the mitigation for the accepted bus-factor-of-one risk on
  bsnes-jg: the code cannot be taken away.

## Alternatives considered

- **A permissive base (ares, ISC)** — not chosen now; would trade away bsnes-jg's fit as an
  actively maintained, SNES-focused, already-a-library core. Documented as the route if a
  permissive fork ever becomes necessary.
- **snes9x** — rejected; non-commercial licence, not OSI-approved, GPL-incompatible.
- **Dynamic linking to dodge the copyleft reach** — rejected; contrary to the single-file
  delivery goal, and the licence is not a constraint worth fighting.
- **OpenAL Soft instead of Steam Audio** — LGPL with no blanket static-linking exception.
  Workable under GPLv3 but more friction than Apache-2.0.
